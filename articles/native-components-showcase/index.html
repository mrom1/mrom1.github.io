Exploring Native Nuxt Content Components

A comprehensive guide to various native Nuxt Content components including collapsible sections, field descriptions, icons, keyboard shortcuts, step-by-step guides, and interactive tabs with code/preview.

November 30, 2025

Unveiling Nuxt Content's Native Component Ecosystem

Nuxt Content provides a rich set of native components that can be directly embedded into your Markdown files, offering powerful ways to structure, explain, and present information. This article explores some of these essential components to enhance your content creation.

Collapsible Component

The collapsible component is useful for hiding detailed information that might otherwise clutter the main content. It's often used for API documentation or complex data structures.

PropDefaultType
namestring
sizemdstring
colorneutralstring

Field Component

The field component is designed for describing properties, options, or parameters, often found in API references or configuration guides. It clearly outlines the name, type, and description of a field.

name
string required
The description can be set as prop or in the default slot with full markdown support. This field represents the unique identifier for an item.

Field Group Component

When you have multiple related fields to describe, the field-group component provides a structured way to list them, making complex configurations easier to digest.

analytics
boolean
Default to false - Enables analytics for your project (coming soon). This flag controls whether usage data is collected.
blob
boolean
Default to false - Enables blob storage to store static assets, such as images, videos and more. Useful for large media files.
cache
boolean
Default to false - Enables cache storage to cache your server route responses or functions using Nitro's cachedEventHandler and cachedFunction. Improves performance.
database
boolean
Default to false - Enables SQL database to store your application's data. Essential for dynamic content and user management.

Icon Component

The icon component allows you to easily embed vector icons from various icon sets (like Lucide, Simple Icons) directly into your content, adding visual cues and enhancing aesthetics.

Keyboard Shortcut Component

The kbd component is perfect for documenting keyboard shortcuts, making it clear to users which keys to press for specific actions.

To open the command palette, press K. This is a common shortcut across many applications.

Steps Component

The steps component is excellent for breaking down complex processes into an easy-to-follow, step-by-step guide. It helps users navigate through instructions efficiently.

Add the Nuxt UI module in your nuxt.config.ts

nuxt.config.ts
export default defineNuxtConfig({
  modules: ['@nuxt/ui']
})

Import Tailwind CSS in your CSS

assets/css/main.css
@import "tailwindcss";

Start your development server

npm run dev

Tabs Component with Code and Preview

This advanced usage of the tabs component allows you to present both the Markdown code for a component and its live preview side-by-side, which is invaluable for demonstrating how components are used and what their output looks like.

::callout{color="green" icon="i-lucide-check-circle"}

By integrating these native components, you can create highly interactive, well-structured, and visually appealing documentation and content within your Nuxt Content application, improving the overall user experience.
